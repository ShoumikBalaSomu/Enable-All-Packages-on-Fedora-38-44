Critical Fixes from Live Fedora 44 Testing

v3.1 patches 11 major issues that break other setup scripts on modern Fedora releases.

Installs `noopenh264` stub when Cisco repo hasn't published F44 `mozilla-openh264` yet.

🔄

FFmpeg Conflict

Uses `dnf swap --allowerasing` to safely replace `ffmpeg-free` with full RPM Fusion `ffmpeg`.

📦

Flatpak Silent Failures

Implements `--or-update`, system-to-user fallback, and pre-install `appstream` refresh.

🔧

🎬

Missing Multimedia Codecs

10-Step Automation

What the Script Does

From system update to final refresh, everything is handled automatically with robust error recovery.

01

System Update

Refreshes package metadata and applies all pending system updates using `sudo dnf update -y`.

02

RPM Fusion

Installs Free & Non-Free repos. Handles `dnf group upgrade` vs `dnf5 groupupdate` automatically.

03

Flathub & Flatpak

Adds Flathub remote (system+user), syncs appstream metadata, installs `gnome-software` with version-aware plugin logic.

04

Snap + Kernel Fixes

Installs `snapd`, un-blacklists `squashfs`, loads kernel module, creates `/snap` symlink, patches all shell PATHs, and waits for `snapd` seeding.

05

OpenH264 Codec

Enables Cisco repo, installs GStreamer plugin. Uses `noopenh264` stub on F44+ with clear fallback instructions.

06

Multimedia Codecs

Swaps ffmpeg-free → full ffmpeg, installs GStreamer suite, `lame`, `x264`, `x265`, `libdvdcss` via tainted repo, and VLC via Flatpak.

07

Google Chrome

Adds official Google Chrome repository securely with GPG key verification and installs `google-chrome-stable`.

08

Visual Studio Code

Imports Microsoft signing key, configures VS Code repo, and installs `code` with proper update channels.

09

Gaming & Media Apps

Installs Heroic Games Launcher & Celluloid via Flatpak with `--or-update` and user fallback. Adds `dnf5-command(copr)` if needed.

10

Final System Refresh

Runs final `dnf update` and `flatpak update` to ensure all newly added repos are fully synchronized. Reminds you to reboot.

Quick Start

How to use

⚠️ Run as a regular user, NOT root. The script will use `sudo` internally when needed.

1

Open Terminal

Launch your terminal on Fedora. Ensure you are logged in as a normal user with `sudo` privileges.

2

Paste & Run

Copy the one-liner below. It fetches the script securely via HTTPS and pipes it directly to bash.

bash <(curl -sL https://raw.githubusercontent.com/ShoumikBalaSomu/Enable-All-Packages-on-Fedora-38-44/main/Thanks_God.sh)
3

Enter Password & Wait

The script will prompt for your sudo password. It runs 10 automated steps (5-15 mins). Sit back and watch it work.

4

Reboot & Verify

Reboot to finalize kernel modules and snap changes. After reboot, test with:

sudo reboot
